D1 Plan & Manage (20-25%)

Key Topics
1.1Select appropriate Foundry Services for each scenario
1.2Create & deploy resources, SDKs, APIs, containers, CI/CD
1.3Monitor, manage costs, secure keys, manage auth
1.4Responsible AI: content moderation, filters, blocklists, prompt shields
Must Know: Multi-service vs single-service resources, key rotation, managed identity, RBAC roles (Cognitive Services User/Contributor), diagnostic logging, cost alerts

D2 Generative AI (15-20%)

Key Topics
2.1Foundry hub/project, prompt flow, RAG, evaluation, SDK
2.2Azure OpenAI: provision, deploy, prompts, DALL-E, multimodal
2.3Parameters, monitoring, tracing, containers, fine-tuning, prompt engineering
Must Know: temperature vs top_p, token limits per model, deployment types (Standard/Provisioned/Global), RAG data source config, evaluation metrics (groundedness, relevance, coherence)

D3 Agentic Solutions (5-10%)

Key Topics
3.1Agent use cases, Foundry Agent Service, Agent Framework, multi-agent orchestration, testing & deployment
Must Know: Agent tools (code interpreter, file search, function calling), thread/run model, multi-agent orchestration patterns, when to use agents vs standard completions

D4 Computer Vision (10-15%)

Key Topics
4.1Image analysis: features, objects, tags, OCR, handwriting
4.2Custom Vision: classification vs detection, training, metrics, publishing
4.3Video Indexer, Spatial Analysis
Must Know: Visual features enum, image size limits, OCR vs Read API, Custom Vision domains (compact vs non-compact), min training images, export formats

D5 NLP Solutions (15-20%)

Key Topics
5.1Text Analytics: sentiment, key phrases, entities, PII, language detection, translation
5.2Speech: STT, TTS, SSML, custom speech, intent/keyword recognition, translation
5.3CLU: intents/entities/utterances, QnA: knowledge bases, multi-turn, custom translator
Must Know: SSML elements (prosody, break, emphasis), CLU workflow, QnA sources, text limits per document (5120 chars), sentiment scoring model, PII categories

D6 Knowledge Mining (15-20%)

Key Topics
6.1AI Search: index, skillset, indexer, queries, knowledge store, vector/semantic search
6.2Document Intelligence: prebuilt models, custom models, composed models
6.3Content Understanding: OCR pipeline, classify, extract entities/tables/images
Must Know: Index field attributes (searchable, filterable, sortable, facetable), query syntax (simple vs Lucene), built-in skillset names, prebuilt Doc Intel model types, composed model limits

Azure AI Portal Gallery — Exact Service Names

Exam Alert: The exam uses "in Foundry Tools" to refer to services accessed through the AI Foundry portal (e.g., "Azure Vision in Foundry Tools"). The underlying APIs and SDKs are identical — it's the portal access path, not a different service.

Video Indexer: Appears in the portal as "Azure Video Indexer". It is NOT part of the Azure AI Services multi-service resource — it requires its own dedicated resource. Supports upload via device (2 GB max) or URL (30 GB max), up to 6 hours duration.
Service Renames: Azure AI Search = formerly "Azure Cognitive Search." Document Intelligence = formerly "Form Recognizer." AI Foundry = formerly "Azure AI Studio." The exam may use either name.

Service Selection Decision Tree

"I need to generate text/code/images" → Azure OpenAI (GPT-4o, DALL-E 3)
"I need to analyze images for objects/tags" → Azure AI Vision (Image Analysis)
"I need to read text from images/PDFs" → Azure AI Vision (Read/OCR) or Document Intelligence
"I need to classify custom images" → Custom Vision
"I need sentiment/entities from text" → Azure AI Language (Text Analytics)
"I need to understand user intent" → CLU (Conversational Language Understanding)
"I need FAQ/QnA bot" → Custom Question Answering
"I need speech-to-text or text-to-speech" → Azure AI Speech
"I need to translate text" → Azure Translator
"I need to search documents with AI" → Azure AI Search + Skillsets
"I need to extract form data" → Document Intelligence (prebuilt or custom)
"I need to moderate content" → Azure Content Safety
"I need an AI agent with tools" → Foundry Agent Service
"I need RAG (search + generate)" → Azure OpenAI + AI Search (On Your Data)

Resource Types & Authentication

Multi-Service vs Single-Service

TypeDetails
Multi-serviceOne endpoint + key for Vision, Language, Speech, etc. Resource type: CognitiveServices
Single-serviceDedicated endpoint per service. Use when you need separate billing, keys, or regions.
Azure OpenAIAlways a separate resource. Requires application approval for some models.

Authentication Methods

MethodWhen to Use
API KeyQuick dev/test. Header: Ocp-Apim-Subscription-Key
Azure AD TokenProduction. RBAC: Cognitive Services User
Managed IdentityService-to-service (no keys in code)
Key Rotation: Two keys per resource. Rotate one while using the other — zero downtime.
