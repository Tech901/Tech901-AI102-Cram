Enrichment Pipeline 3-Stage Flow

1. Document Cracking

Indexer opens files, extracts text content and normalized_images array. Supports PDF, Office docs, HTML, JSON, CSV, plain text, and images.

2. Enrichment (Skillset)

Skills execute against the enrichment tree — a hierarchical in-memory document. Each skill reads input nodes and writes output nodes. Skills chain via /document/... paths.

3. Output Projection

Results flow to two destinations:

  • Search index — via outputFieldMappings (enriched fields) and fieldMappings (raw fields)
  • Knowledge store — via projection definitions (tables, objects, files)
Common Gotcha: fieldMappings map source → index BEFORE enrichment. outputFieldMappings map enrichment tree → index AFTER skillset runs. Don't confuse them!
Pipeline summary: Crack → Enrich → Project

Semantic Search Re-Ranking

Configuration Fields

PropertyPurpose
titleFieldDocument title (single field)
contentFieldsMain content (ordered by priority)
keywordFieldsKeywords / tags

How It Works

Re-ranks top 50 BM25 results using deep learning models from Bing. Does NOT replace keyword search — it enhances it.

Captions: Extractive with highlights (bolded key phrases)
Answers: Extractive, max 10 per query
Score: Returns @search.rerankerScore (0–4 scale)

Exam Tip: Semantic search requires Standard tier or higher. It re-ranks, not re-retrieves — if a document isn't in the initial BM25 top 50, semantic won't find it.

Search Client — Semantic Query SDK

Python 
# pip install azure-search-documents
from azure.core.credentials import AzureKeyCredential
from azure.search.documents import SearchClient
from azure.search.documents.models import (
    VectorizedQuery, QueryType, QueryCaptionType, QueryAnswerType
)

client = SearchClient(
    endpoint=f"https://{service_name}.search.windows.net",
    index_name="memphis-311-index",
    credential=AzureKeyCredential(api_key))

# Hybrid + semantic search
results = client.search(
    search_text="pothole repair on Beale Street",
    search_fields=["title", "content"],
    select=["id", "title", "content", "category"],
    filter="category eq 'infrastructure'",
    top=10,
    include_total_count=True,
    query_type=QueryType.SEMANTIC,
    semantic_configuration_name="my-semantic-config",
    query_caption=QueryCaptionType.EXTRACTIVE,
    query_answer=QueryAnswerType.EXTRACTIVE,
    vector_queries=[
        VectorizedQuery(
            vector=embedding,  # 1536 floats from OpenAI
            k_nearest_neighbors=50,
            fields="contentVector"
        )
    ])

print(f"Total: {results.get_count()}")
for result in results:
    print(f"{result['title']} (score: {result['@search.score']})")
C# 
// dotnet add package Azure.Search.Documents
using Azure;
using Azure.Search.Documents;
using Azure.Search.Documents.Models;

var client = new SearchClient(
    new Uri($"https://{serviceName}.search.windows.net"),
    "memphis-311-index",
    new AzureKeyCredential(apiKey));

// Simple search with filter
var options = new SearchOptions
{
    Filter = "category eq 'infrastructure'",
    Size = 10,
    IncludeTotalCount = true,
    Select = { "id", "title", "content", "category" },
    SearchFields = { "title", "content" },
    QueryType = SearchQueryType.Semantic,
    SemanticSearch = new SemanticSearchOptions
    {
        SemanticConfigurationName = "my-semantic-config",
        QueryCaption = new QueryCaption(QueryCaptionType.Extractive),
        QueryAnswer = new QueryAnswer(QueryAnswerType.Extractive)
    }
};

SearchResults<SearchDocument> results =
    client.Search<SearchDocument>("pothole repair on Beale Street", options);

Console.WriteLine($"Total: {results.TotalCount}");
foreach (SearchResult<SearchDocument> result in results.GetResults())
{
    Console.WriteLine($"{result.Document["title"]} (score: {result.Score})");
}

Flow: Keyword BM25 + Vector ANN run in parallel → RRF merges results → Semantic re-ranks top 50 → captions/answers extracted.
Exam Tip: vectorQueries[].k controls how many vector neighbors to retrieve. $top controls final results returned. They are independent.
Remember: You must generate the embedding vector client-side (e.g., via Azure OpenAI Embeddings API) before sending the search request. The search service does NOT generate embeddings at query time unless you configure an integrated vectorizer.

Exam Gotchas & Decision Guide Must Know

fieldMappings vs outputFieldMappings: fieldMappings run BEFORE enrichment (raw source → index). outputFieldMappings run AFTER enrichment (enrichment tree → index). This is the #1 skillset question.
queryType values: simple (default), full (Lucene), semantic. You can combine semantic with either simple or full.
Scoring profiles: Custom relevance boosting using functions (freshness, magnitude, distance, tag). Applied BEFORE semantic re-ranking.
$filter string comparison: Filters use exact match for Edm.String — no full-text analysis. Use search.ismatch() in a filter for analyzed text matching.
Indexer field mappings for Blob: metadata_storage_path (Base64-encoded URI) is commonly used as the document key. Use base64Encode mapping function.
Incremental enrichment: Enable caching by attaching a storage account to the skillset. Caches enrichment tree so unchanged documents skip re-processing.
Private endpoints: Use shared private link for indexer connections to data sources behind firewalls. Max indexer run time extends to 24 hours with private endpoints.
CORS: Must be configured on the search service to allow browser-based JavaScript queries. Set allowedOrigins in service configuration.
