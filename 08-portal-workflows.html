Portal Workflows & Step-by-Step Guides

AI-102 Cram Sheet — Portal navigation for every major Azure AI service

1. Create an Azure AI Services Resource

Portal: portal.azure.com

Steps

1
Click Create a resource in the Azure portal
2
Search for Azure AI services and select it
3
Select your Subscription and Resource Group (or create new)
4
Choose a Region (affects available services and latency)
5
Enter a globally unique Resource name
6
Select Pricing tier — typically S0 (Standard)
7
Click Review + Create, then Create

Key Settings

SettingNotes
Resource nameMust be globally unique across Azure
RegionDetermines which AI services are available
Pricing tierF0 (free, 1 per subscription) or S0 (standard)
After creation: Go to Keys and Endpoint to get credentials. You get two keys for zero-downtime key rotation.

2. Deploy an Azure OpenAI Model

Portal: ai.azure.com (Azure AI Foundry)

Steps

1
Open your project in Azure AI Foundry
2
Navigate to Models + endpoints
3
Click Deploy model → Deploy base model
4
Select model (e.g., gpt-4o, gpt-4o-mini, text-embedding-ada-002)
5
Configure a Deployment name (this is used in API calls)
6
Set Deployment type: Standard (pay-per-token) or Provisioned (reserved throughput)
7
Set Tokens-per-minute (TPM) rate limit
8
Click Deploy

Key Settings

SettingNotes
Deployment nameReferenced in model parameter of API calls
TPM quotaRate limit per deployment; shared across subscription + region
Content filterDefault applied automatically; can assign custom filter
Exam tip: The deployment name is NOT the same as the model name. API calls use the deployment name.

3. Configure Content Filters

Portal: ai.azure.com (Azure AI Foundry)

Steps

1
Open your Foundry resource and go to Safety + security → Content filters
2
Click Create content filter and give it a name
3
Configure Input filters: set severity level for each of the 4 categories (Hate, Sexual, Violence, Self-harm)
4
Configure Output filters with the same 4 categories
5
Enable or disable Prompt Shields (jailbreak detection) and Protected Material detection
6
Optionally add Blocklists for custom blocked terms
7
Save the filter, then associate it with a deployment

Severity Levels

LevelBehavior
LowBlocks low, medium, and high severity
MediumBlocks medium and high severity
HighBlocks only high severity
Important: Turning filters completely OFF requires Limited Access approval from Microsoft. This is a common exam question.

4. Create an Azure AI Search Index

Portal: portal.azure.com

Manual Index Creation

1
Create an Azure AI Search resource and choose a tier (Free, Basic, Standard, etc.)
2
Go to the resource → Indexes → Add index
3
Define fields: name, type (Edm.String, Edm.Int32, etc.), and attributes
4
Set one field as the Key (must be Edm.String)
5
Configure per-field attributes: Searchable, Filterable, Sortable, Facetable, Retrievable
6
Click Create

Import Data Wizard (Alternative)

1
Connect to data source (Blob Storage, SQL, Cosmos DB, etc.)
2
Add cognitive skills — attach a skillset for AI enrichment
3
Customize target index — adjust fields and attributes
4
Create indexer to populate the index
Exam tip: The key field must be Edm.String. Every index requires exactly one key field.

5. Set Up a Search Indexer with Skillset

Portal: portal.azure.com → AI Search resource

Steps

1
In your AI Search resource, go to Indexers → New indexer
2
Select a Data source (Azure Blob Storage, SQL Database, Cosmos DB, Table Storage)
3
Attach an AI Services resource for billing (required for built-in skills beyond 20 free enrichments/day)
4
Configure skills: OCR, Key Phrase Extraction, Entity Recognition, Language Detection, Sentiment, Image Analysis, etc.
5
Map enriched output fields to your target index fields
6
Set indexer schedule (once, hourly, daily, or custom)
7
Run the indexer

Skillset Pipeline

Flow: Data Source → Indexer → Skillset (enrichment) → Index
ComponentPurpose
Data sourceWhere raw documents live
SkillsetAI enrichment pipeline (built-in + custom skills)
Output field mappingsConnect enriched fields to index schema
Knowledge storeOptional: persist enrichments to Blob/Table storage
Tip: Use the Import data wizard for guided setup, or the REST API / SDK for full programmatic control.

6. Create a CLU Project (Conversational Language Understanding)

Portal: language.cognitive.azure.com (Language Studio)

Steps

1
Open Language Studio and select your Language resource
2
Choose Conversational Language Understanding
3
Create new project — set name and primary language
4
Add intents (e.g., ReportPothole, CheckStatus)
5
Add entities (e.g., Location, IssueType)
6
Add utterances to each intent and label entities within them
7
Train the model (Standard or Advanced training)
8
Evaluate results: review Precision, Recall, and F1 scores
9
Deploy the model to a named deployment slot
10
Test with sample queries in the testing pane
Minimum requirements: At least 1 intent defined, entities labeled in utterances, model must be trained before deployment.
Exam tip: CLU replaces the older LUIS service. LUIS is now deprecated — migrate to CLU.

7. Create a Custom Question Answering Project

Portal: language.cognitive.azure.com (Language Studio)

Steps

1
Open Language Studio → Custom Question Answering
2
Create new project (requires an AI Search resource for the knowledge base)
3
Add sources: URLs, uploaded files (PDF, DOCX, TXT), or pre-built chit-chat
4
Review auto-extracted QnA pairs and edit as needed
5
Add alternate phrasings to improve matching
6
Add follow-up prompts to create multi-turn conversations
7
Optionally add metadata tags for filtering answers
8
Test in the test pane, then Deploy

Source Limits

Source TypeLimit
URL crawl depthUp to 20 levels deep
File upload sizeUp to 25 MB per PDF
Chit-chatPre-built personality sets (Professional, Friendly, etc.)
Tip: Custom QA requires an Azure AI Search resource as its backing store. Plan for this when provisioning.

8. Deploy a Custom Vision Model

Portal: customvision.ai

Steps

1
Create project: choose Classification or Object Detection
2
Choose a domain (General, Food, Landmarks, Retail, or Compact variants for export)
3
Upload images and tag/label them (draw bounding boxes for detection)
4
Train: Quick Training (faster) or Advanced Training (set time budget)
5
Review metrics: Precision, Recall, mAP (mean Average Precision for detection)
6
Set probability threshold to tune precision vs. recall
7
Publish the iteration to a prediction resource
8
Get the Prediction URL and Prediction Key

Minimum Image Requirements

Project TypeMinimum Images per Tag
Classification5 images per tag
Object Detection15 images per tag
Exam tip: Use a Compact domain if you need to export the model to run on edge devices (ONNX, TensorFlow, CoreML).

9. Create a Document Intelligence Model

Portal: documentintelligence.ai.azure.com (Document Intelligence Studio)

Prebuilt Models

1
Open Document Intelligence Studio and select your resource
2
Choose a prebuilt model (Invoice, Receipt, ID Document, Business Card, W-2, etc.)
3
Upload a sample document and analyze

Custom Models

1
Select Custom model → create new project
2
Upload training documents to a Blob Storage container
3
Label fields in each document using the labeling tool
4
Train the model (minimum 5 labeled documents recommended)
5
Test with new documents, then deploy

Composed Models

Composed model: Combine multiple custom models into a single endpoint. The service automatically routes documents to the best sub-model. Up to 500 models on S0 tier.
Exam tip: Know the difference between prebuilt (zero training), custom template (fixed layout), and custom neural (variable layout) models.

10. Set Up RAG (On Your Data)

Portal: ai.azure.com (Azure AI Foundry — Chat Playground)

Steps

1
Open the Chat playground in Azure AI Foundry
2
Click Add your data
3
Select data source: existing AI Search index, upload files, or URL
4
Configure index field mappings: content field, title field, filename field, URL field
5
Set search type: Keyword, Semantic, Vector, or Hybrid
6
Set Strictness (1–5): higher = fewer but more relevant results
7
Set Retrieved documents count (3–20)
8
Save and close, then chat with your grounded model

Search Type Comparison

Search TypeHow It WorksBest For
KeywordBM25 term matchingExact term lookups
SemanticReranks keyword results with deep learningNatural language queries
VectorEmbedding similarity searchConceptual / cross-language queries
Hybrid + SemanticKeyword + Vector + Semantic rerankingBest overall relevance
Best practice: Use Hybrid + Semantic ranking for the best relevance. This combines BM25, vector similarity, and neural reranking.

11. Create a Foundry Agent

Portal: ai.azure.com (Azure AI Foundry)
Formula: Agent = Model + Instructions + Tools

Steps

1
Open your project → Agents → Create agent
2
Give the agent a name
3
Select the model (e.g., gpt-4o)
4
Write instructions (system prompt defining the agent's behavior)
5
Add tools: Code Interpreter, File Search, Azure AI Search, custom Functions
6
Upload files if using File Search or Code Interpreter
7
Test in the playground and review execution traces
8
Evaluate with test datasets, then Publish

Available Tools

ToolPurposeWhen to Use
Code InterpreterRun Python in a sandboxData analysis, charts, math, file conversion
File SearchVector search over uploaded filesRAG over PDFs/docs the user provides
Azure AI SearchQuery your search indexesGround on enterprise data (hybrid/semantic)
Web SearchReal-time Bing groundingCurrent events, live information
Function CallingCustom functions (you define + handle)Call your APIs, databases, or business logic
Azure FunctionsServerless backend actionsAuto-executed custom code (no client handling)
Image GenerationDALL-E in-conversationCreate images from text descriptions
Browser AutomationNavigate websites via natural languageWeb scraping, form filling (preview)
SharePointChat over SharePoint docsEnterprise document Q&A (preview)
Function Calling vs Azure Functions: With Function Calling, the agent returns the function name + args and you execute it client-side. With Azure Functions, the agent calls the function directly — no client code needed.

12. Monitor AI Resources

Portal: portal.azure.com

Enable Diagnostic Logging

1
Go to your AI resource → Monitoring section → Diagnostic settings
2
Click Add diagnostic setting
3
Select which logs and metrics to collect
4
Choose destination: Send to Log Analytics workspace
5
Save the diagnostic setting

Create Alert Rules

1
Go to Alerts → Create alert rule
2
Select a metric (e.g., Total Calls, Total Errors, Latency)
3
Set condition and threshold (e.g., errors > 10 in 5 min)
4
Configure an action group (email, SMS, webhook, Logic App, etc.)

Key Metrics to Monitor

MetricWhat It Tells You
Total CallsRequest volume / usage patterns
Successful CallsHealthy request count
Total ErrorsFailures (4xx, 5xx, throttling)
LatencyResponse time (P50, P99)
Token UsageConsumption for OpenAI models

13. Manage Keys and Authentication

Portal: portal.azure.com

Key-Based Authentication

1
Go to your resource → Keys and Endpoint
2
Copy Key 1 or Key 2 (both are valid simultaneously)
3
To rotate: click Regenerate Key 1, update apps to use Key 2, then regenerate Key 2 later

Managed Identity (Recommended for Production)

1
Go to your compute resource (App Service, VM, etc.) → Identity
2
Set System assigned status to On → Save
3
Go to the AI resource → Access control (IAM) → Add role assignment
4
Assign Cognitive Services User role to the managed identity

Authentication Comparison

MethodUse CaseSecurity
API KeysDevelopment, quick startRotate regularly, never commit to code
Managed IdentityProduction Azure-to-AzureNo credentials to manage
Azure AD TokenUser-delegated accessFine-grained RBAC
Exam tip: Two keys exist for zero-downtime rotation. Regenerating Key 1 does NOT affect Key 2.

14. Configure Network Security

Portal: portal.azure.com

Network Access Options

1
Go to your AI resource → Networking
2
Choose access level:
OptionDescription
All networksPublic internet access (default, least secure)
Selected networksAllow specific VNets/IPs only
DisabledPrivate endpoint only (most secure)

Set Up Private Endpoint

1
Go to Networking → Private endpoint connections → Create
2
Select the target sub-resource (e.g., account)
3
Link to a VNet and subnet
4
Configure Private DNS zone integration for name resolution
5
Create the private endpoint

Add VNet Rules

1
Under Selected networks, click Add existing virtual network
2
Select allowed VNets and subnets
3
Optionally add specific IP address ranges
Best practice: Use Private Endpoints for production. Traffic stays on the Microsoft backbone network and never traverses the public internet.
