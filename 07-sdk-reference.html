Authentication Patterns

1. API Key (AzureKeyCredential)

Python 
from azure.core.credentials import AzureKeyCredential

credential = AzureKeyCredential("your-api-key")
client = TextAnalyticsClient(
    endpoint="https://myresource.cognitiveservices.azure.com/",
    credential=credential
)
C# 
using Azure;
using Azure.AI.TextAnalytics;

var credential = new AzureKeyCredential("your-api-key");
var client = new TextAnalyticsClient(
    new Uri("https://myresource.cognitiveservices.azure.com/"),
    credential
);

2. Azure AD (DefaultAzureCredential)

Python 
from azure.identity import DefaultAzureCredential

credential = DefaultAzureCredential()
client = TextAnalyticsClient(
    endpoint="https://myresource.cognitiveservices.azure.com/",
    credential=credential
)
C# 
using Azure.Identity;
using Azure.AI.TextAnalytics;

var credential = new DefaultAzureCredential();
var client = new TextAnalyticsClient(
    new Uri("https://myresource.cognitiveservices.azure.com/"),
    credential
);

3. AzureOpenAI Client

Python 
from openai import AzureOpenAI

client = AzureOpenAI(
    api_key="your-api-key",
    azure_endpoint="https://myresource.openai.azure.com/",
    api_version="2024-10-21"
)
C# 
using Azure;
using Azure.AI.OpenAI;

var client = new AzureOpenAIClient(
    new Uri("https://myresource.openai.azure.com/"),
    new AzureKeyCredential("your-api-key")
);

REST API Headers

Most AI Services: Ocp-Apim-Subscription-Key: <key>
Azure OpenAI: api-key: <key>
Exam Tip: The header name difference is a common exam question. OpenAI uses api-key; all other Cognitive Services use Ocp-Apim-Subscription-Key.

Pattern to remember: Most services use .cognitiveservices.azure.com. The exceptions are OpenAI (.openai.azure.com), Translator (global endpoint), AI Search (.search.windows.net), and Speech (region-based websocket).
Multi-service resource: A single Azure AI Services resource provides one .cognitiveservices.azure.com endpoint for Vision, Language, Speech, and Content Safety. One key works across all of them.

Azure OpenAI — Chat Completion

Python 
from openai import AzureOpenAI

client = AzureOpenAI(
    azure_endpoint=endpoint,
    api_key=key,
    api_version="2024-10-21"
)

response = client.chat.completions.create(
    model="gpt-4o",  # deployment name
    messages=[
        {"role": "system", "content": "You are helpful."},
        {"role": "user", "content": "Hello"}
    ],
    temperature=0.7,
    max_tokens=800
)
print(response.choices[0].message.content)
C# 
using Azure;
using Azure.AI.OpenAI;
using OpenAI.Chat;

var azureClient = new AzureOpenAIClient(
    new Uri(endpoint),
    new AzureKeyCredential(key)
);
var chatClient = azureClient.GetChatClient("gpt-4o");

ChatCompletion response = await chatClient
    .CompleteChatAsync(
        new ChatMessage[]
        {
            new SystemChatMessage("You are helpful."),
            new UserChatMessage("Hello")
        },
        new ChatCompletionOptions
        {
            Temperature = 0.7f,
            MaxOutputTokenCount = 800
        }
    );

Console.WriteLine(response.Content[0].Text);
cURL 
curl -X POST "{endpoint}/openai/deployments/gpt-4o/chat/completions?api-version=2024-10-21" \
  -H "api-key: {key}" \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      {"role": "system", "content": "You are helpful."},
      {"role": "user", "content": "Hello"}
    ],
    "temperature": 0.7,
    "max_tokens": 800
  }'

Batch operations: All Text Analytics methods accept a list of documents. Results return in the same order. Each document can be up to 5,120 characters; batch limit is 10 documents per call (25 for sentiment).

CLU — Analyze Conversation

Python 
from azure.ai.language.conversations import (
    ConversationAnalysisClient
)
from azure.core.credentials import AzureKeyCredential

client = ConversationAnalysisClient(
    endpoint, AzureKeyCredential(key)
)

result = client.analyze_conversation(
    task={
        "kind": "Conversation",
        "analysisInput": {
            "conversationItem": {
                "id": "1",
                "participantId": "1",
                "text": "Turn on the lights"
            }
        },
        "parameters": {
            "projectName": "myProject",
            "deploymentName": "production"
        }
    }
)

prediction = result["result"]["prediction"]
top_intent = prediction["topIntent"]
confidence = prediction["intents"][0]["confidenceScore"]

for entity in prediction["entities"]:
    print(entity["category"], entity["text"])
C# 
using Azure;
using Azure.AI.Language.Conversations;
using System.Text.Json;

var client = new ConversationAnalysisClient(
    new Uri(endpoint), new AzureKeyCredential(key));

var data = new
{
    kind = "Conversation",
    analysisInput = new
    {
        conversationItem = new
        {
            id = "1",
            participantId = "1",
            text = "Turn on the lights"
        }
    },
    parameters = new
    {
        projectName = "myProject",
        deploymentName = "production"
    }
};

Response<BinaryData> response = await client
    .AnalyzeConversationAsync(
        RequestContent.Create(BinaryData
            .FromObjectAsJson(data)));

using var doc = JsonDocument.Parse(
    response.Value.ToMemory());
var prediction = doc.RootElement
    .GetProperty("result")
    .GetProperty("prediction");
string topIntent = prediction
    .GetProperty("topIntent").GetString();

foreach (var entity in prediction
    .GetProperty("entities").EnumerateArray())
{
    Console.WriteLine(
        $"{entity.GetProperty("category")} " +
        $"{entity.GetProperty("text")}");
}
CLU task structure: The request must include kind, analysisInput.conversationItem (with id, participantId, text), and parameters (projectName, deploymentName). This nested dict structure is exam-tested.

Question Answering

client = QuestionAnsweringClient(

using Azure.AI.Language.QuestionAnswering;

var client = new QuestionAnsweringClient(
    new Uri(endpoint), new AzureKeyCredential(key));

QuestionAnsweringProject project =
    new("myKB", "production");

Response<AnswersResult> response = await client
    .GetAnswersAsync("What is Azure?", project);

{
    Console.WriteLine(answer.Answer);

QnA vs. CLU: Question Answering returns answers from a knowledge base. CLU returns intents and entities from utterances. Both use the Language resource but different client classes.

Image Analysis

Python 
from azure.ai.vision.imageanalysis import (
    ImageAnalysisClient
)
from azure.ai.vision.imageanalysis.models import (
    VisualFeatures
)
from azure.core.credentials import AzureKeyCredential

client = ImageAnalysisClient(
    endpoint, AzureKeyCredential(key)

result = client.analyze_from_url(

        VisualFeatures.TAGS,

print(result.caption.text, result.caption.confidence)
for tag in result.tags.list:

for block in result.read.blocks:

var client = new ImageAnalysisClient(

ImageAnalysisResult result = await client
    .AnalyzeAsync(

        VisualFeatures.Caption |
        VisualFeatures.Tags |

Console.WriteLine(

Document Intelligence

Python 
from azure.ai.formrecognizer import (
    DocumentAnalysisClient
)
from azure.core.credentials import AzureKeyCredential

client = DocumentAnalysisClient(
    endpoint, AzureKeyCredential(key)
)

poller = client.begin_analyze_document(
    "prebuilt-invoice",
    document=open("invoice.pdf", "rb")
)
result = poller.result()

for doc in result.documents:
    print(doc.doc_type)
    for name, field in doc.fields.items():
        print(name, field.value, field.confidence)
C# 
using Azure;
using Azure.AI.FormRecognizer.DocumentAnalysis;

var client = new DocumentAnalysisClient(
    new Uri(endpoint), new AzureKeyCredential(key));

using var stream = File.OpenRead("invoice.pdf");
var operation = await client
    .AnalyzeDocumentAsync(
        WaitUntil.Completed,
        "prebuilt-invoice",
        stream);

AnalyzeResult result = operation.Value;

foreach (var doc in result.Documents)
{
    Console.WriteLine(doc.DocumentType);
    foreach (var (name, field) in doc.Fields)
    {
        Console.WriteLine(
            $"{name} {field.Content} " +
            $"{field.Confidence}");
    }
}
cURL 
curl -X POST "{endpoint}/formrecognizer/documentModels/prebuilt-invoice:analyze?api-version=2024-11-30" \
  -H "Ocp-Apim-Subscription-Key: {key}" \
  -H "Content-Type: application/pdf" \
  --data-binary @invoice.pdf
Prebuilt models: prebuilt-invoice, prebuilt-receipt, prebuilt-idDocument, prebuilt-businessCard, prebuilt-layout, prebuilt-read. The model ID string is the first argument to begin_analyze_document().
Long-running operation: begin_analyze_document() returns a poller. Call poller.result() to block until complete. Same pattern used for custom model training.

Speech-to-Text

Python 
import azure.cognitiveservices.speech as speechsdk

config = speechsdk.SpeechConfig(
    subscription=key, region=region
)

# Optional: set source language
config.speech_recognition_language = "en-US"

recognizer = speechsdk.SpeechRecognizer(
    speech_config=config
)
result = recognizer.recognize_once()

if result.reason == speechsdk.ResultReason.RecognizedSpeech:
    print(result.text)
elif result.reason == speechsdk.ResultReason.NoMatch:
    print("No speech recognized")
C# 
using Microsoft.CognitiveServices.Speech;

var config = SpeechConfig.FromSubscription(key, region);

// Optional: set source language
config.SpeechRecognitionLanguage = "en-US";

using var recognizer = new SpeechRecognizer(config);
var result = await recognizer.RecognizeOnceAsync();

if (result.Reason == ResultReason.RecognizedSpeech)
    Console.WriteLine(result.Text);
else if (result.Reason == ResultReason.NoMatch)
    Console.WriteLine("No speech recognized");

AI Search

Python 
from azure.search.documents import SearchClient
from azure.core.credentials import AzureKeyCredential

client = SearchClient(
    endpoint,
    "index-name",
    AzureKeyCredential(key)
)

results = client.search(
    search_text="luxury hotel",
    filter="Rating gt 4",
    select=["HotelName", "Rating"],
    top=10,
    order_by=["Rating desc"]
)

for r in results:
    print(r["HotelName"], r["@search.score"])
C# 
using Azure;
using Azure.Search.Documents;

var client = new SearchClient(
    new Uri(endpoint),
    "index-name",
    new AzureKeyCredential(key)
);

var options = new SearchOptions
{
    Filter = "Rating gt 4",
    Select = { "HotelName", "Rating" },
    Size = 10,
    OrderBy = { "Rating desc" }
};

SearchResults<SearchDocument> results =
    await client.SearchAsync<SearchDocument>(
        "luxury hotel", options);

await foreach (var r in results.GetResultsAsync())
{
    Console.WriteLine(
        $"{r.Document["HotelName"]} " +
        $"{r.Score}");
}
cURL 
curl -X POST "{endpoint}/indexes/index-name/docs/search?api-version=2024-07-01" \
  -H "api-key: {key}" \
  -H "Content-Type: application/json" \
  -d '{
    "search": "luxury hotel",
    "filter": "Rating gt 4",
    "select": "HotelName,Rating",
    "top": 10,
    "orderby": "Rating desc"
  }'
SearchClient constructor: Takes 3 positional args: endpoint, index name, credential. The index name is the second arg — not a keyword on the search call.
OData filters: Use filter= for exact/range queries: Rating gt 4, Location eq 'Memphis', Tags/any(t: t eq 'wifi'). Filterable fields must be marked filterable in the index schema.

Common RBAC Roles for Azure AI Services

RoleAccess LevelExam Notes
Cognitive Services UserCall APIs (read)Most common role for app service principals and developers
Cognitive Services ContributorCreate/manage resources (no key access)Can manage but NOT call APIs — common exam trap
Cognitive Services OpenAI UserCall OpenAI APIsScoped to OpenAI only; use for least-privilege access
Cognitive Services OpenAI ContributorFine-tune + deploy modelsRequired for model deployment and fine-tuning operations
Search Index Data ReaderQuery search indexesRead-only access to index data; cannot modify index schema
Search Index Data ContributorManage index contentUpload, merge, delete documents; cannot create/delete indexes
Search Service ContributorManage search serviceFull control over service config, indexes, indexers, skillsets
Exam Tip: "Contributor" roles for Cognitive Services grant management-plane access (create resources, view keys) but do NOT grant data-plane access (calling APIs). To call APIs, you need the "User" role. This is the opposite of what most people expect.
