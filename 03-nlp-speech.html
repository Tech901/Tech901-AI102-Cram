Text Analytics Capabilities

Sentiment Analysis

Returns positive, negative, neutral, or mixed with confidence scores 0–1. Operates at both document level and sentence level.

Opinion Mining: Enable show_opinion_mining=True for aspect-based sentiment — extracts targets (e.g., “food”) and assessments (e.g., “delicious”) with per-aspect sentiment.

Key Phrase Extraction

Returns a list of main talking points from the text. Useful for summarization and indexing.

Named Entity Recognition (NER)

Identifies entities with categories: Person, Location, Organization, DateTime, Quantity, URL, Email, IPAddress, PhoneNumber, Event, Product, Skill.

Entity Linking

Disambiguates entities and links them to Wikipedia articles. Returns URL, data source, and a Bing entity ID.

PII Detection

Categories: Person, Address, Phone, Email, SSN, CreditCard, BankAccount, PassportNumber, DriversLicense, USIndividualTaxpayerIdentification.

redactedText: Returns the original text with PII replaced by ***. Set domain="phi" to detect only Protected Health Information.

Language Detection

Detects 120+ languages. Returns ISO 639-1 language code + confidence score (0–1). Unknown language returns (Unknown) with code unknown.

CLU Conversational Language Understanding

Core Concepts

ConceptDescription
IntentsUser goals/actions the model classifies (e.g., BookFlight, GetWeather)
EntitiesKey data extracted from utterances (e.g., city, date, number)
UtterancesExample user inputs labeled with intents & entities for training

Entity Components

ComponentHow it works
LearnedML-based, trained from labeled utterances
ListExact match with synonyms (e.g., “NYC” → “New York City”)
PrebuiltSystem entities: number, datetime, temperature, etc.
RegexPattern match (e.g., [A-Z]{2}\d{4} for flight codes)
Multi-component entities: A single entity can combine multiple components. The final value is resolved by priority: List > Regex > Prebuilt > Learned.

Training Modes

ModeSpeedLanguagesUse Case
StandardFasterEnglish onlyQuick iteration
AdvancedSlowerMultilingualProduction, non-English

Workflow

1. Define Schema → Create intents and entity definitions
2. Label Utterances → Tag example text with intents + entity spans
3. Train → Standard or Advanced mode
4. Evaluate → Precision, Recall, F1 score per intent/entity
5. Deploy → Assign trained model to a deployment slot
6. Predict → Send user text, receive top intent + extracted entities
Watch Out: Authoring limit is 500 chars but prediction allows 1,000 chars. Exam may test both values.

Speech Services Overview

Speech-to-Text (STT)

ModeDetails
Real-timeStreaming via SDK or WebSocket. Interim + final results.
BatchAsync processing from Azure Blob Storage. For pre-recorded audio.
Default audio format: WAV, 16 kHz sample rate, 16-bit, mono PCM.
SDK compressed formats: MP3, OPUS/OGG, FLAC, AAC (requires GStreamer for some).

Text-to-Speech (TTS)

400+ neural voices across 140+ languages and variants. Output at 24 kHz or 48 kHz.

Max audio per synthesis: 10 minutes. Use Batch Synthesis API for longer content.

Speaking Styles

Available on select neural voices (e.g., en-US-JennyNeural):

cheerful angry sad whispering newscast customerservice

30+ styles total. Controlled via SSML <mstts:express-as>.

Style degree: Range 0.01 to 2 (default 1). Values >1 amplify the style; <1 soften it.
Watch Out: Not all voices support all styles. Check the voice gallery for style availability per voice.

Complete SSML Example

<speak version="1.0"
       xmlns="http://www.w3.org/2001/10/synthesis"
       xmlns:mstts="https://www.w3.org/2001/mstts"
       xml:lang="en-US">

  <voice name="en-US-JennyNeural">

    <!-- Speaking style with degree -->
    <mstts:express-as style="cheerful" styledegree="1.5">
      Welcome to the Memphis 311 service!
    </mstts:express-as>

    <break strength="strong" />

    <!-- Prosody control -->
    <prosody rate="slow" pitch="low" volume="loud">
      Your case number is
      <say-as interpret-as="characters">AB1234</say-as>.
    </prosody>

    <break time="750ms" />

    <!-- Date and telephone -->
    Your appointment is on
    <say-as interpret-as="date" format="mdy">03/25/2026</say-as>.

    Please call us at
    <say-as interpret-as="telephone">+19015551234</say-as>.

    <!-- Emphasis -->
    <emphasis level="strong">Thank you</emphasis> for using 311.

  </voice>
</speak>
Exam Tip: The <speak> root element MUST have version="1.0" and xml:lang. For Microsoft styles, add the xmlns:mstts namespace. Forgetting any of these is a common exam trap.
Watch Out: <break time> max is 20,000 ms (20 seconds). styledegree only works inside <mstts:express-as>, not on <prosody>.

Speaker Recognition

Verification vs Identification

FeatureVerification (1:1)Identification (1:N)
PurposeConfirm “Are you who you claim?”Determine “Who is speaking?”
Text-dependentYes (passphrase required)Not supported
Text-independentYesYes
Max candidates1 (comparison target)50
Accept threshold: Score >= 0.5 means accepted. Below 0.5 means rejected.

Text-Dependent Verification

Passphrase: User must speak a specific phrase (e.g., “My voice is my passport, verify me”). Requires 3 enrollment samples of the same passphrase.

Text-Independent

No passphrase required. User speaks freely. Needs at least 20 seconds of speech for enrollment. Verification needs 4+ seconds of speech.

SDK Pattern

Python 
import azure.cognitiveservices.speech as speechsdk

# Speech config
config = speechsdk.SpeechConfig(
    subscription=SPEECH_KEY,
    region=SPEECH_REGION
)

# Speech-to-Text
recognizer = speechsdk.SpeechRecognizer(
    speech_config=config,
    audio_config=audio_config
)
result = recognizer.recognize_once()

# Text-to-Speech
synthesizer = speechsdk.SpeechSynthesizer(
    speech_config=config
)
# Plain text
result = synthesizer.speak_text("Hello Memphis!")
# SSML
result = synthesizer.speak_ssml(ssml_string)
C# 
// dotnet add package Microsoft.CognitiveServices.Speech
using Microsoft.CognitiveServices.Speech;
using Microsoft.CognitiveServices.Speech.Audio;

// Speech config
var config = SpeechConfig.FromSubscription(speechKey, speechRegion);

// Speech-to-Text
using var audioConfig = AudioConfig.FromDefaultMicrophoneInput();
using var recognizer = new SpeechRecognizer(config, audioConfig);
var result = await recognizer.RecognizeOnceAsync();
Console.WriteLine($"Recognized: {result.Text}");

// Text-to-Speech
using var synthesizer = new SpeechSynthesizer(config);
// Plain text
var ttsResult = await synthesizer.SpeakTextAsync("Hello Memphis!");
// SSML
var ssmlResult = await synthesizer.SpeakSsmlAsync(ssmlString);
Exam Tip: Speaker recognition uses the Speech SDK, not a separate service. The key difference: text-dependent is more accurate but less flexible; text-independent works with any speech but needs more audio.

Common Exam Traps

Text Analytics

5,120 chars: Per-document limit for synchronous calls. Longer text must be chunked.
PII domain="phi": Only detects health-related PII. Without it, all PII categories are returned.

CLU

500 vs 1,000 chars: Authoring limit is 500, but prediction accepts up to 1,000 characters.
Standard training: English-only. If the question mentions multilingual, the answer is Advanced.

Speech & SSML

10-minute TTS limit: Single synthesis max. Use Batch Synthesis for audiobooks/podcasts.
break time max: 20,000 ms. Anything higher is invalid SSML.
styledegree on prosody: This does NOT work. styledegree is only valid on <mstts:express-as>.
