API Chat Completions Endpoint

POST Endpoint: {endpoint}/openai/deployments/{deployment-id}/chat/completions?api-version=2024-10-21

SDK Usage

Python 
# pip install openai
from openai import AzureOpenAI

client = AzureOpenAI(
    azure_endpoint=endpoint,
    api_key=api_key,
    api_version="2024-10-21"
)

response = client.chat.completions.create(
    model="gpt-4o",  # deployment name
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Hello!"}
    ],
    temperature=0.7,
    max_tokens=800
)

print(response.choices[0].message.content)
print(f"Finish reason: {response.choices[0].finish_reason}")
print(f"Usage: {response.usage.prompt_tokens} + {response.usage.completion_tokens}")
C# 
// dotnet add package Azure.AI.OpenAI
using Azure;
using Azure.AI.OpenAI;
using OpenAI.Chat;

var client = new AzureOpenAIClient(
    new Uri(endpoint),
    new AzureKeyCredential(apiKey));

var chatClient = client.GetChatClient("gpt-4o");  // deployment name

ChatCompletion completion = chatClient.CompleteChat(
    new ChatMessage[]
    {
        new SystemChatMessage("You are a helpful assistant."),
        new UserChatMessage("Hello!")
    },
    new ChatCompletionOptions
    {
        Temperature = 0.7f,
        MaxOutputTokenCount = 800,
        TopP = 1f,
        FrequencyPenalty = 0f,
        PresencePenalty = 0f
    });

Console.WriteLine(completion.Content[0].Text);
Console.WriteLine($"Finish reason: {completion.FinishReason}");
Console.WriteLine($"Usage: {completion.Usage.InputTokenCount} + {completion.Usage.OutputTokenCount}");

Message Roles

RolePurpose
systemSets behavior, personality, constraints. Processed first. Not for o-series models.
developerReplaces system for o1/o1-mini reasoning models.
userUser prompts. Can be string or array (for multimodal: text + image_url).
assistantModel responses & few-shot examples. May contain tool_calls.
toolFunction call results. Must include tool_call_id matching the request.

API All Request Parameters

ParameterTypeRangeDefaultDescription
temperaturefloat0–21Controls randomness. Higher = more creative.
top_pfloat0–11Nucleus sampling. Change ONE of temp OR top_p, not both.
max_tokensint1–model_maxvariesMaximum tokens in the completion.
frequency_penaltyfloat-2 to 20Penalize tokens by how often they appeared. Reduces repetition.
presence_penaltyfloat-2 to 20Penalize tokens that appeared at all. Encourages new topics.
stopstring/arrayup to 4nullStop sequences — generation halts when encountered.
nint1+1Number of completions to generate per request.
streambool—falseServer-Sent Events (SSE) streaming.
seedint—nullDeterministic sampling — same seed + params = same output.
response_formatobject——text, json_object, or json_schema (structured output).
toolsarraymax 128nullFunction definitions for tool/function calling.
tool_choicestring/obj—autonone, auto, required, or specific function name.
Exam Trap: frequency_penalty scales by count (repeat 10x = 10x penalty). presence_penalty is flat (appeared = penalized). Know the difference.

finish_reason Values

ValueMeaning
stopModel completed naturally or hit a stop sequence.
lengthHit max_tokens limit — output was truncated.
content_filterCompletion blocked by content filtering.
tool_callsModel wants to call one or more functions.
Exam Tip: usage.prompt_tokens + usage.completion_tokens = usage.total_tokens. Billing is based on total tokens.

Prompting Prompt Engineering Techniques

Shot Techniques

TechniqueDescription
Zero-shotNo examples. Relies on model's training. Good for simple tasks.
One-shotOne example in the prompt. Sets format and tone.
Few-shotMultiple examples (3–5). Best for consistent format & style.
Chain-of-thought"Let's think step by step." Forces reasoning before answering.

Delimiter Patterns

DelimiterUsage
###Section separators

System Message Best Practices

  • Define the persona and task clearly
  • Specify output format (JSON, markdown, etc.)
  • Set constraints: what NOT to do
  • Include safety guardrails and fallback behavior
  • Place few-shot examples after system instructions

Temperature Tuning Guide

Use CaseTemperature
Code generation, factual Q&A0.0 – 0.3
Summarization, classification0.3 – 0.7

Structured Output

response_format: Use {"type": "json_object"} to force valid JSON output. Must mention "JSON" in the prompt or you get an error. For strict schemas use {"type": "json_schema", "json_schema": {...}}.

RequirementValue

EncodingUTF-8 with BOM
Max file size512 MB
Minimum examples10 (hundreds recommended)
Max training time720 hours

Fine-Tuning Workflow

1. Prepare — Create JSONL with messages array (system, user, assistant turns)
2. Upload — Upload training file via Files API (purpose: "fine-tune")
3. Train — Create fine-tuning job. Set hyperparameters (epochs, learning rate, batch size)
4. Deploy — Deploy the custom model to an endpoint
5. Use — Call the deployment like any other Chat Completions endpoint
Required RBAC: Cognitive Services OpenAI Contributor role. Regular Cognitive Services User is NOT enough for fine-tuning.

JSONL Training Format

{"messages": [
  {"role": "system", "content": "You classify support tickets."},
  {"role": "user", "content": "My internet is down"},
  {"role": "assistant", "content": "Category: Network"}
]}

Safety Content Filtering

Four Content Categories

CategoryCovers
HateHate speech, discrimination, slurs
SexualSexually explicit content
ViolenceViolent acts, graphic content
Self-HarmSelf-injury, suicide

Severity Levels

LevelValueDescription
Safe0No harmful content detected
Low2Minor concerns
Medium4Moderate severity
High6Severe content

Default Behavior

Default threshold: Medium — blocks Medium (4) + High (6) severity. Allows Safe (0) + Low (2).
